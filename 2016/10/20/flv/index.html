FLV file format

工作中需要熟悉视频流方面的知识，其中就包括FLV的格式，以下是对学习内容的一些记录。

The FLV file format

从Flash Player 6开始，Flash Player能够在RTMP连接上和Adobe Flash Media Server交换音视频数据。一种给Flash Media Server和Flash Player clients提供数据的方式是使用FLV格式的文件。Flash Player 7之后，Flash播放器能够直接播放由HTTP传过来的FLV数据，MIME类型为video/x-flv。

一个FLV文件同时编码音频和视频流，它们的编码方式和SWF文件是相同的。

FLV文件中每一个类型的tag构成了一个单独的流。一个FLV文件中不会同时超过一个音频流和一个视频流，也就是最多一路音频流，一路视频流。

一个FLV由header和body两部分组成。

The FLV header

所有的FLV文件在开头都有一个header，它的格式如下：

field type comment
Signature 1 byte ‘F’
Signature 1 byte ‘L’
Signature 1 byte ‘V’
Version 1 byte 文件版本号
TypeFlagsReserved 5 bit 保留字段，全0
TypeFlagsAudio 1 bit 是否存在音频流
TypeFlagsReserved 1 bit 保留字段，全0
TypeFlagsVideo 1 bit 是否存在视频流
DataOffset 4 byte body部分字节偏移量

对于version为1的FLV，DataOffset始终为9，即header的大小为9个字节。

The FLV file body

FLV header之后就是一连串的back-pointers、tags交替排列，格式如下所示：

Field type comment
PreviousTagSize0 4 byte 0，因为前一个tag不存在
Tag1 FLVTAG 第一个tag
PreviousTagSize1 4 byte 前一个tag的大小
Tag2 FLVTAG 第二个tag
… … …
TagN FLVTAG 最后一个tag
PreviousTagSizeN 4 byte 最后一个tag的大小

FLV tags

FLV tags的格式如下：

field type comment
TagType 1 byte tag类型，8=audio，9=video，18=script data
DataSize 3 byte Data字段长度
Timestamp 3 byte 相对第一个tag的时间毫秒数
TimestampExtended 1 byte 和Timestamp组成PTS
StreamID 3 byte 0
Data 见下方解释 tag body

TagType ~ StreamID构成一个tag header。Data字段的类型由TagType字段的值来决定：

  • TagType = 8时，Data = audio tag
  • TagType = 9时，Data = video tag
  • TagType = 18时，Data = script tag

Audio Tag

Audio Tag格式如下。

Field Type Comment
SoundFormat 4 bit 音频编码类型
SoundRate 2 bit 采样率
SoundSize 1 bit 采样精度
SoundType 1 bit 音频类型
SoundData payload 音频数据

Audio tags的前一个字节为音频相关参数，后面接着真正的音频数据。重点看当SoundFormat=10时，SoundData为AAC类型数据。

AAC

Field Type Comment
AACPacketType 1 byte 0: AAC sequence header；1: AAC raw
Data byte[] 见下方解释

Data字段：有效载荷，当AACPacketType=0时，Data为AudioSpecificConfig；当AACPacketType=1时，Data为原始AAC帧。AAC sequence header即AudioSpecificConfig，包含了更详细的音频信息。一般FLV文件的第一个audio tag的类型为AAC sequence header。

Video Tag

Video Tag格式如下。

Field Type Comment
FrameType 4 bit 帧类型
CodecID 4 bit 编码类型
VideoData payload 视频帧载荷或1 byte

FrameType：

  • 1：keyframe
  • 2：inter frame
  • 3：disposable inter frame
  • 4：generated keyframe
  • 5：video info/command frame

CodecID：

  • 1：JPEG
  • 2：Sorenson H.263
  • 3：Screen video
  • 4：On2 VP6
  • 5：On2 VP6 with alpha channel
  • 6：Screen video version 2
  • 7：AVC

VideoData：

  • CodecID = 2时，VideoData = H263VIDEOPACKET
  • CodecID = 3时，VideoData = SCREENVIDEOPACKET
  • CodecID = 4时，VideoData = VP6FLVVIDEOPACKET
  • CodecID = 5时，VideoData = VP6FLVALPHAVIDEOPACKET
  • CodecID = 6时，VideoData = SCREENV2VIDEOPACKET
  • CodecID = 7时，VideoData = AVCVIDEOPACKET

当FrameType = 5时，VideoData为一个byte的信息：

  • 0：客户端寻找帧序列的起始位置
  • 1：客户端寻找帧序列的结束为止

AVCVIDEOPACKET

这种类型的packet携带AVC编码类型的数据帧，格式如下：

Field Type Comment
AVCPacketType 1 byte 0：AVC sequence header；1：AVC NALU；2：AVC end of sequence
CompositionTime 3 byte 见下方解释
Data byte array 见下方解释

CompositionTime：

  • 当AVCPacketType = 1，此字段表示Composition time offset
  • 否则，此字段为0

Data：

  • 当AVCPacketType = 0，Data = AVCDecoderConfigurationRecord。AVCDecoderConfigurationRecord包含着是H.264解码相关较重要的sps和pps信息（sps和pps放在了同一个tag中），再给AVC解码器送数据流之前一定要把sps和pps信息送出，否则的话解码器不能正常解码。而且在解码器stop之后再次start之前，如seek、快进快退状态切换等，都需要重新送一遍sps和pps的信息。AVCDecoderConfigurationRecord在FLV文件中一般情况也是出现一次，也就是第一个video tag。
  • 当AVCPacketType = 1，Data = One or more NALUs (can be individual slices per FLV packets; that is, full frames are not strictly required)，NALU（network abstract layer unit）简单理解就是一帧视频数据。
  • 当AVCPacketType = 2，Data为空

为了防止SPS,PPS数据丢失，有些编码器会在每个IDR帧之前重复发SPS,PPS。这些SPS其实是一样的。但也不排除有些变态的编码器前后的SPS会不同，毕竟标准容许这样做。这样就需要首先遍历一边h264码流，将其中不同的SPS、PPS提起出来，先记录下来，然后再统一写到FLV。

Script Tag

Script tag又被称为Metadata tag，包含了一些视频或音频数据的元信息。通常该类型Tag会跟在File Header后面作为第一个Tag出现，而且只有一个。Script tag中包含两个AMF包。两个AMF的排列如下。

AMF1

value Type Comment
0x02 1 byte 0x02表示字符串
0x000A 2 byte 一般总是字符串“onMetaData”的长度
onMetaData string 6F,6E,4D,65,74,61,44,61,74,61

AMF2

value Type Comment
0x08 1 byte 0x08表示数组
array size 4 byte 数组元素个数
[duration:xxx, width:xxx, height:xxx, …] array 音视频数据元信息

AMF2的数组中包含一些KV对，常见类型有如下几种。

key 含义
duration 时长
width 视频宽度
height 视频高度
videodatarate 视频码率
framerate 视频帧率
videocodecid 视频编码方式
audiosamplerate 音频采样率
audiosamplesize 音频采样精度
stereo 是否为立体声
audiocodecid 音频编码方式
filesize 文件大小

关于GOP

GOP是一个图像组，在FLV格式中没有GOP这个概念，它是人为建立出来的一个概念，将两个I帧之间的所有帧都认为是一个GOP。由于一个单独的GOP有独立播放的能力，因此在直播中常常将它作为一个传输单元进行发送，并且丢弃一个GOP也不会影响正常播放。

通常一个GOP中既包含video tag，也包含audio tag，但这没关系，只要认为两个I帧tag之间的数据，都认为是一个GOP即可。
